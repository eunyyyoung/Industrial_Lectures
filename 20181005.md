오픈소스 활동, 깃헙, Vue.js - 이선협 오픈소스 개발자
======
오픈소스는 1980년대 리처드 스톨만의 자유 소프트웨어 운동으로 시작되었다.
지금에 이르러서는 오픈소스 없이는 개발하기 힘든 세상이 되었다. <br>
오픈소스를 할 때, 내 실력으로 해도 되는지, 내 코드를 비웃진 않을지, 내 코드가 옳은 코드인지 고민 할 필요없다. 
오히려 오픈소스에 기여하며 코드 리뷰도 받고 다른 사람의 코드를 보며 실력을 향상시킬 수 있다. <br>
* 오픈소스를 시작할 때 잘하면 좋은 세가지
	- git
	- 영어 : 영어로 대화할 경우가 많다.
	- 프로그래밍 능력 <br>

필수는 아니고 오픈소스에 기여 하다보면 점점 는다. <br>
기존 오픈소스에 기여하기 위해서는 여러가지 방법이 있는데 그  중 멋진 것은 해당 오픈소스 프로젝트에 기능을 추가하거나 버그를 수정하는 것이지만
초심자는 우선 쉬운 것부터 오타 수정, 문법 수정 등이 있다.
먼저 Issue나 Pull Request 목록에 해당 문제를 수정한 사람이 있는지 확인하고, 없다면 수정해서 Pull Request를 날린다.
https://github.com/octocat/Spoon-Knife 에서 연습할 수 있다.
다음 스텝으로는 번역이 있다. 오픈소스 라이브러리의 가이드 문서 혹은 스타일 가이드 문서를 번역하면 좋다. <br>
커뮤니티가 귀찮다면 온라인(페이스북 그룹, Slack)을 통해 커뮤니티를 시작하는 것을 추천한다.
온라인에서 충분히 커뮤니티의 분위기를 파악했다면 오프라인 모임을 시도해보는 것도 좋다.
